Changing Origin Remote URL for GitHub Repo

Changing the origin remote URL for a GitHub repository involves a few simple steps. Here's a step-by-step guide:

1. Check Current Remote URL

Before making any changes, it's a good practice to verify the current remote URL. You can do this by navigating to your repository directory in your terminal and running the following command: 

git remote -v

2. Update Remote URL

To update the origin remote URL, you'll use the git remote set-url command followed by the name of the remote (usually origin) and the new URL. For example: 

git remote set-url origin <new_remote_url>

Replace <new_remote_url> with the new URL you want to set for the origin remote.

3. Verify Changes

After updating the remote URL, it's a good idea to verify that the changes have been applied correctly. You can do this by running the git remote -v command again: 

git remote -v

This will display the updated remote URLs.

Example:

Let's say you want to change the origin remote URL from https://github.com/old_username/repository.git to https://github.com/new_username/repository.git.

You would execute the following commands:

git remote set-url origin https://github.com/new_username/repository.git

Then, to verify the change:

git remote -v

Additional Notes:

  • Make sure you have the necessary permissions to push to the new repository URL.
  • If you're using SSH URLs instead of HTTPS, you'd use the SSH URL format (git@github.com:new_username/repository.git) instead of the HTTPS format.

That's it! You've successfully changed the origin remote URL for your GitHub repository.
